DEFCONFIGURATION: a configuration tool for Common Lisp

This is the home of the portable DEFCONFIGURATION tool for Common Lisp, a "configuration tool" in the spirit of UNIX configure and imake tools.

DEFCONFIGURATION current version is 1.0.

Does Common Lisp need a "configuration" tool?

In order to set up a complex Common Lisp application, a programmer must make provisions to actually specify where and how to load several initialization files and libraries. This task is usually accomplished via a set of script-like facilities, most of the time gourped in files named load-application etc.

DEFCONFIGURATION utilities have appeared in several implementations over the years, starting from Symbolics implementations, to the somewhat portable version appearing in PCL, to the proprietary versions present in Harlequin LispWorks and Franz ACL.

The story of this specific version of DEFCONFIGURATION starts with the implementation by Mark Kantrowitz, which appeared in the Lisp Utilities collection at CMU and which is available in the well known AI.Repository.

Mark Kantrowitz's DEFCONFIGURATION is a good compromise of notation and functionality and it has the great advantage of being available on all the major Common Lisp implementations. Hence its technical importance.

Two problems hampered DEFCONFIGURATION along the years.

  1. The time the author devoted to further development diminished drastically (people have to get real jobs);

  2. the distribution license was considered too restrictive (in the commercial sense) by some users.

Meanwhile, during 1998/1999, the X3J13 committee started working again. The question of a DEFCONFIGURATION for Common Lisp came up immediately as one of the hot topics. DEFCONFIGURATION was proposed as a solution, but the two aforementioned problems were mentioned as stumbling blocks.

To try to patch this set of circumstances, the author of this page(s) decided to ask Mark Kantrowitz to change the copyright notice and the distribution license. After a few exchanged emails, Mark Kantrowitz kindly allowed me to "take over" DEFCONFIGURATION and to address the two problems miring its acceptance as the de facto DEFCONFIGURATION standard for Common Lisp.

DEFCONFIGURATION features

  • Declarative system construction (dependencies are on the CL.ENVIRONMENT package facilities).

  • Runs on all major Common Lisp implementations (Franz Allegro, Harlequin Common Lisp, Genera, CMUCL, CLisp, MCL, KCL and derivatives).

  • Integrates with the provide and require functionality, plus with a plethora of DEFSYSTEM tools, most notably, MK:DEFSYSTEM.
